Detailed Description

Common part of a node in XList.

_XListNode_base would record and care concurrent usage of data.

Increment usageNO by "_inc".

Parameters
_incnumber of increment, default is "1".

We don't use any lock to increment usageNO. This beacause of atomic nature of "usageNO". Also wait operation on "usageNO" in "waitForUsage" would be done at deletion time, when no one could access him to increment "usageNO", so "usageNO" never been increased at deletion time.

◆ n_lock

pthread_mutex_t pparam::_XListNode_base::n_lock
protected

Lock to manage decrement changes of "usageNO".

Also use to manage access next/prev nodes from "this". "n_lock" would be locked before (next/prev)pointer read, and would be released after "access" to node pointed with (next/prev) pointer. So between pointer read and node access, next/prev pointers couldn't change.
